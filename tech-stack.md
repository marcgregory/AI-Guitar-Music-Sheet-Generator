# AI Guitar Music Sheet Generator / MusicStudio - Recommended Open-Source Tech Stack

## Frontend (Client-Side)
- **Framework**: React (with TypeScript) - For building responsive, interactive UIs with strong ecosystem support
- **UI Library**: MUI (Material-UI) or Ant Design - For accessible, responsive components with dark/light mode support
- **Music Notation/TAB Rendering**: 
  - VexFlow - Open-source library for rendering standard notation and guitar tablature
  - Alternatively: ABCJS for standard notation + custom tab renderer for guitar-specific needs
- **State Management**: Zustand or Redux Toolkit - For managing transcription state, user preferences, and project data
- **Build Tool**: Vite - For fast development builds and optimized production bundles
- **Audio Playback**: Web Audio API (with Howler.js wrapper if needed) - For synchronized playback and tempo controls
- **YouTube Integration**: Embed YouTube iframe API for preview (processing happens on backend)
- **Export Handling**: 
  - jsPDF or html2pdf.js for client-side PDF generation
  - FileSaver.js for downloading generated files

## Backend (Server-Side)
- **Language**: Python 3.11 - Current backend/runtime target
- **Web Framework**: FastAPI - For high-performance async APIs with automatic OpenAPI docs
- **Audio Processing Core**:
  - Librosa - Fundamental for audio analysis (chroma, beat tracking, feature extraction)
  - Essentia - Advanced audio/music analysis (key, tempo, chord detection)
  - Demucs - For MVP source separation using default stems: vocals, drums, bass, and other
  - Future specialist separation models - For true guitar, lead/rhythm guitar, piano, or other instrument-specific stems
  - Spotify Basic Pitch (open-source model) or CREPE - For pitch detection
  - Custom chord recognition pipeline using Librosa features + lightweight ML model (scikit-learn/TensorFlow)
- **YouTube Audio Download**: yt-dlp (actively maintained fork of youtube-dl) - For extracting audio from user-provided YouTube links
- **Database**: PostgreSQL (with SQLAlchemy ORM) - For user accounts, project metadata, and saved transcriptions
- **Authentication**: 
  - FastAPI Users (built on Firebase Auth concepts) or custom JWT implementation
  - Optional: Integrate with OpenID Connect providers (Google, GitHub) via python-social-auth
- **File Storage**:
  - Cloudinary for durable uploaded audio and generated outputs in the MVP
  - Store Cloudinary `secure_url` and `public_id` for original audio, separated stem audio, MIDI files, and TAB files
  - Railway local filesystem only as temporary scratch space during worker processing
- **Lyrics Generation**: faster-whisper for vocal stems, tracked with `lyrics_generation_status` separately from the main audio `processing_status`
- **Task Queue / Orchestration**: FastAPI job records plus Modal dispatch/callback coordination. `AUDIO_PROCESSING_MODE=modal` is the hosted MVP path. Redis/Celery is local development fallback only and should run with `--concurrency=1` for very short files.
- **API Documentation**: Swagger UI (auto-generated by FastAPI) + ReDoc

## DevOps & Infrastructure
- **Containerization**: Docker + Docker Compose - For consistent development/deployment environments
- **MVP Hosting**: Vercel or current host for frontend, Railway/Render for FastAPI/PostgreSQL API/controller, Cloudinary for durable file storage, and Modal GPU for AI/audio processing
- **External Worker Testing**: Kaggle/manual notebooks may be used for experiments, but they are not 24/7 production infrastructure and are not the documented processing mode
- **Orchestration**: Kubernetes (for production) or Docker Swarm (simpler alternative)
- **CI/CD**: GitHub Actions - For automated testing, building, and deployment
- **Monitoring**: Prometheus + Grafana - For tracking system performance and processing times
- **Logging**: ELK Stack (Elasticsearch, Logstash, Kibana) or Loki/Promtail/Grafana
- **Reverse Proxy**: Nginx or Traefik - For SSL termination and routing

## Additional Considerations
- **PDF Generation**: WeasyPrint (HTML/CSS to PDF) or ReportLab - For high-quality sheet music exports
- **MIDI/MusicXML Generation**: 
  - music21 - Comprehensive music notation toolkit that supports MIDI/MusicXML
  - Alternatively: mido for MIDI + custom MusicXML generation
- **Testing**: 
  - Frontend: Jest + React Testing Library
  - Backend: Pytest + TestClient (FastAPI)
  - Audio processing: Custom test suites with known audio samples
- **Accessibility**: 
  - axe-core for automated accessibility testing
  - Manual testing with screen readers (NVDA, VoiceOver)
- **Internationalization**: i18next (frontend) + Python gettext (backend) - For future localization

## Why This Stack?
1. **Open Source Commitment**: Every component is actively maintained open-source software with permissive licenses (MIT, Apache, GPL-compatible)
2. **Audio Processing Maturity**: Python ecosystem has the most advanced open-source audio/MIR (Music Information Retrieval) libraries
3. **MVP Stability**: FastAPI orchestrates jobs while Modal/serverless GPU handles the heavy Demucs workload. Local Celery remains available for development fallback without making Railway the production AI worker.
4. **Cost Efficiency**: Selective stem processing, duplicate detection, Cloudinary-backed durable storage, and serverless GPU processing minimize repeated Demucs runs, unnecessary queue jobs, Railway CPU/RAM use, disk pressure, storage cost, and processing time.
5. **Community Support**: Large communities around React, Python audio libraries, and PostgreSQL
6. **Extensibility**: Modular design allows swapping components (e.g., trying different AI models)
7. **Compliance Focus**: Designed to process user-provided content only with automatic deletion of raw audio

## Optional Enhancements (Future Phases)
- **Browser-based AI**: TensorFlow.js/ONNX.js for experimental client-side pitch detection
- **Advanced Instrument Support**: multiple selected stems, AlphaTab or VexFlow rendering, better quantization, chord grouping, fingering optimization, MusicXML/GP-like export, manual correction editor, specialist models, and advanced Songsterr-like multi-track tabs
- **Machine Learning Tracking**: MLflow for model experiment tracking
- **Real-time Collaboration**: Yjs or Socket.IO for multi-user editing
- **Progressive Web App**: Workbox for offline capabilities and installability
